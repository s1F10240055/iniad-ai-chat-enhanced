# INIAD AI Chat Enhanced 提案書

## 1. 提案概要

| 項目 | 内容 |
| ---------- | ---------------------------------------------------------------------------------- | |
| 想定ユーザ | INIAD（東洋大学 情報連携学部）の学生 |
| 提供価値 | INIAD の講義・学習文脈に即した質問を、専用チャットUIから素早く行える学習支援ツール |
| 形態 | Electron デスクトップアプリ |
| 開発体制 | 提案者 3名（全員が設計・実装を担当） |

本提案は、既存の「Slack 経由で INIAD API を叩き、OpenAI API のモデルと会話できる」体験を出発点とし、学習用途に特化したチャット製品へ拡張するものである。

## 2. 背景と課題

### 2.1 現状

INIAD では、Slack 上から INIAD の API を経由して OpenAI API の各種モデル（GPT-4o 等）とチャットできる環境が提供されている。学生は日常的にこの機能を利用しているが、利用体験には以下の課題がある。

### 2.2 課題

| No. | 課題                                        | 影響                                                                  |
| --- | ------------------------------------------- | --------------------------------------------------------------------- |
| 1   | INIAD の講義資料（MOOCs）を自動参照できない | 講義固有の内容について質問しても、LLMの一般知識に基づく回答になりがち |
| 2   | 最新情報や外部情報の取得ができない          | 時事ネタや最新トピックについて回答できない                            |
| 3   | 回答の根拠が不明確                          | 学生が回答の正確性を検証しづらい                                      |
| 4   | Slack UI がチャット用途に最適化されていない | スレッド形式の制約、履歴管理の不便さ                                  |
| 5   | モデル切り替えや文脈管理が煩雑              | 学習に集中できない操作コスト                                          |

このため、学生が「講義内容を一発で聞く」という本来の利用シーンに対して、**精度・利便性・信頼性**の全てが不足している。

## 3. 目的

1. **講義資料に基づく高精度回答**: INIAD 学生が MOOCs 講義資料に基づいた回答を短時間で得られる環境を提供する
2. **参照元の明示**: 回答の根拠（参照元資料・リンク）を提示し、学習上の再確認を容易にする
3. **学習に最適化された UX**: Slack 依存から脱却し、チャットに特化した UI/UX を実現する
4. **拡張可能性**: ウェブ情報取得等の将来拡張に耐えうるアーキテクチャを最初から採用する

## 4. 技術選定と理由

| 技術                | 用途                         | 選定理由                                                                                                                                                                                                                                                                                  |
| ------------------- | ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Electron**        | デスクトップアプリ基盤       | Node.js エコシステムを活用可能。MCP クライアント（stdio/SSE 通信）を Main プロセスで容易に統合。既存プロジェクト資産（electron-forge）の活用。クロスプラットフォーム対応。                                                                                                                |
| **INIAD-MOOCs-MCP** | 講義資料検索サーバ           | MCP（Model Context Protocol）規格に準拠。INIAD の MOOCs ページから講義資料を検索・取得可能。MCP エコシステムの標準的なツール連携が可能。                                                                                                                                                  |
| **INIAD API**       | LLM 推論                     | INIAD が契約している OpenAI API 互換の学内基盤。Chat Completion / Embeddings / Text Completion / Models / Moderations を提供。APIキーは INIAD Slack の「GPT-5.4」ボットで `apikey issue` を実行して発行。INIAD 活動関連の範囲で学生が無料利用可能（コストは発生するため無駄遣いは禁止）。 |
| **google-sr**       | ウェブ検索（スクレイピング） | Google検索結果をスクレイピングするTypeScriptネイティブライブラリ。APIキー不要。構造化された結果（タイトル・URL・スニペット）を取得可能。学習用途の補助検索として十分な性能。                                                                                                              |

### なぜ Tauri ではなく Electron か

- MCP クライアントライブラリ（`@modelcontextprotocol/sdk`）の Node.js 版が最も成熟している
- チームメンバーの TypeScript スキルセットとの親和性（型安全性によるバグ削減）
- electron-forge によるパッケージング・配布パイプラインが既に構築済み
- 将来的な Browser View 拡張（学内サイト埋め込み等）が容易

## 5. MVP（最小実用製品）

### スコープ

MVP は **「MOOCs 講義資料に基づく回答ができるチャット」** に集中する。

| 機能                      | 優先度   | 内容                                                                         |
| ------------------------- | -------- | ---------------------------------------------------------------------------- |
| チャット送受信            | **必須** | ユーザと LLM の基本的な対話                                                  |
| 設定画面                  | **必須** | APIキー・モデル・MOOCs認証情報をGUIから設定・保存                            |
| MOOCs 資料検索・参照      | **必須** | INIAD-MOOCs-MCP 経由で講義資料を検索し、回答に組み込む                       |
| 参照元表示                | **必須** | 回答の根拠となった資料のタイトル・リンクを表示                               |
| フォールバック            | **必須** | MCP 未接続時でもプレーン会話にフォールバック                                 |
| エラー表示                | **必須** | 接続失敗・資料未取得時のユーザガイド                                         |
| Web検索（スクレイピング） | 推奨     | Google等の検索エンジンスクレイピングによる一般情報の補助検索（時間があれば） |

### スコープ外（初期リリース）

- 成績管理・履修登録などの操作自動化
- 学外有料データソースの統合
- 個人最適化推薦（履修履歴ベース）
- 音声入力・画像入力
- モデルファインチューニング

## 8. リスクと対策

| リスク                     | 確率 | 影響 | 対策                                                                           |
| -------------------------- | ---- | ---- | ------------------------------------------------------------------------------ |
| MOOCs 資料の更新追従遅れ   | 中   | 中   | 起動時・定期インデックス更新ジョブの導入                                       |
| INIAD API の仕様変更       | 低   | 高   | API クライアントの抽象化レイヤーで吸収                                         |
| MCP Server の不安定性      | 中   | 中   | フォールバック機構（プレーン会話）の実装                                       |
| ハルシネーション           | 高   | 高   | 参照元提示の徹底、根拠不十分時は「情報なし」を明示                             |
| 利用規約抵触               | 低   | 高   | API優先、スクレイピング時は必ず規約確認                                        |
| 認証情報漏洩               | 低   | 高   | SettingsStore管理、Renderer露出禁止、マスク表示、ログ秘匿化                    |
| メンバー欠席（病欠・就活） | 中   | 高   | タスクの細粒度分割、キー情報のWiki常時更新、全員が設計・実装を把握する共有体制 |
| MCP検索精度の限界          | 中   | 中   | キーワードベース簡易マッチング（MVP）、将来はセマンティック検索へ移行          |

## 11. 成功定義

MVP 成功の定義：

1. INIAD の講義に関する質問に対し、MOOCs 資料を根拠とした回答が得られる
2. 回答には常に参照元（資料タイトル・リンク）が表示される
3. MCP が利用できない状況でも、プレーンなチャットとして機能する
4. 設定画面から API キー・モデル・MOOCs 認証情報を GUI で管理できる
5. 主要な利用シナリオでクラッシュしない
6. エラー発生時にユーザが理解可能なメッセージと次アクションが提示される
7. MCP連携時のエンドツーエンド応答が10秒以内で完了する
